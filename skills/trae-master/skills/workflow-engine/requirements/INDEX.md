## How to Start a New Requirement
## 如何启动新需求

When a new requirement arrives:
当收到新需求时：

1. **Create Requirement ID**: Use next sequential ID (e.g., REQ-2026-001)
2. **创建需求ID**: 使用下一个顺序ID（例如REQ-2026-001）
3. **Create Directory**: In `skills/workflow-engine/requirements/active/`, create `REQ-YYYY-NNN/` folder
4. **创建目录**: 在 `skills/workflow-engine/requirements/active/` 下创建 `REQ-YYYY-NNN/` 文件夹
5. **Fill Templates**: Copy templates from `skills/workflow-engine/requirements/templates/` into the requirement directory
6. **填写模板**: 将 `skills/workflow-engine/requirements/templates/` 中的模板复制到需求目录
7. **Start Timeline**: Begin recording timestamps in `timeline.md`
8. **开始时间线**: 在 `timeline.md` 中开始记录时间戳

## Recording Rules
## 记录规则

- **Record ONLY after user confirms OK**
- **只有在用户确认 OK 后才记录**
- **Boss approval AND User confirmation are REQUIRED** before creating a phase record or writing to context-log
- **必须经过Boss审核通过且用户确认后**，方可创建并持久化阶段记录或写入 context-log
- **All timestamps MUST be precise** (YYYY-MM-DD HH:MM:SS)
- **所有时间戳必须精确**（YYYY-MM-DD HH:MM:SS）
- **Do NOT modify** completed requirement records
- **不得修改**已完成的需求记录
